# План создания сервиса для бизнеса на Авито

## P0. Базовая архитектура кабинета

### 1. Модель совместного кабинета
- Создать сущность "Кабинет / Workspace / Organization"
- Все основные сущности привязывать к кабинету, а не напрямую к пользователю
- Продумать базовые роли:
  - Владелец
  - Администратор
  - Менеджер
  - Аналитик
  - Только просмотр
- Продумать права доступа:
  - Управление объявлениями
  - Управление задачами
  - Управление аккаунтами Авито
  - Управление рекламой
  - Работа с чатами
  - Просмотр аналитики
  - Управление пользователями

### 2. Пользователи и приглашения
- Регистрация пользователя
- Создание первого кабинета
- Приглашение менеджера по email
- Принятие приглашения
- Удаление пользователя из кабинета
- Смена роли пользователя
- Блокировка пользователя внутри кабинета

### 3. Проекты
- Создать сущность "Проект"
- Привязать задачи, объявления, аккаунты и аналитику к проектам
- Добавить базовые поля проекта:
  - Название
  - Описание
  - Категории
  - Города
  - Ответственные менеджеры
- Сделать список проектов
- Сделать создание/редактирование проекта

### 4. Журнал действий
- Логировать важные действия:
  - Кто создал задачу
  - Кто изменил задачу
  - Кто запустил публикацию
  - Кто подключил аккаунт
  - Кто изменил настройки рекламы
- Показывать историю действий в карточке задачи/объявления
- Это особенно важно для командной работы

---

## P1. MVP: задачи на создание рандомных объявлений

### 5. Довести текущую форму задачи до рабочей версии
- Разделить форму на логические блоки:
  - Основная информация
  - Категория
  - Заголовки
  - Описания
  - Изображения
  - Адреса
  - Опции категории
  - Контакты
  - Расписание
  - Настройки генерации
- Добавить строгую валидацию формы
- Сделать понятные ошибки валидации
- Добавить режим черновика
- Добавить режим предпросмотра перед сохранением

### 6. Модель задачи генерации объявлений
- Создать сущность "Задача"
- Поля задачи:
  - Проект
  - Название
  - Категория
  - Статус
  - Количество объявлений
  - Заголовки
  - Описания
  - Изображения
  - Адреса
  - Опции
  - Расписание
  - Автор задачи
  - Ответственный менеджер
- Статусы задачи:
  - Черновик
  - Готова к запуску
  - Запущена
  - На паузе
  - Завершена
  - Завершена с ошибками
  - Отменена

### 7. Генерация объявлений из задачи
- Создать механизм генерации вариантов
- Рандомизировать:
  - Заголовок
  - Описание
  - Фото
  - Адрес
  - Контакты
  - Дополнительные параметры
- Исключать полные дубли
- Сохранять результат генерации
- Показывать список сгенерированных объявлений до публикации
- Позволить вручную отредактировать сгенерированное объявление

### 8. Безопасный запуск задачи
- Добавить кнопку "Сгенерировать"
- Добавить кнопку "Запустить публикацию"
- Перед запуском показывать summary:
  - Сколько объявлений будет создано
  - В каких городах
  - В какой категории
  - От какого аккаунта
  - Когда публикация начнется
- Добавить подтверждение запуска
- Добавить отмену задачи до публикации

### 9. История выполнения задачи
- Показывать прогресс выполнения
- Показывать успешные публикации
- Показывать ошибки
- Показывать причину ошибки по каждому объявлению
- Добавить повторный запуск только для ошибочных объявлений

---

## P2. Интеграция с аккаунтами Авито

### 10. Подключение аккаунтов Авито
- Создать сущность "Аккаунт Авито"
- Привязать аккаунты к кабинету и проектам
- Добавить подключение через официальный доступ/API, если доступен
- Хранить статус подключения:
  - Активен
  - Требует переподключения
  - Ошибка авторизации
  - Недостаточно прав
  - Заблокирован/недоступен

### 11. Account Health
- Сделать страницу здоровья аккаунтов
- Показывать:
  - Статус авторизации
  - Доступные права
  - Ошибки API
  - Баланс, если доступен
  - Ограничения
  - Проблемы с публикациями
- Добавить уведомления о критических проблемах

### 12. Адаптер публикации
- Не привязывать бизнес-логику напрямую к API Авито
- Сделать внутренний слой:
  - Подготовка объявления
  - Валидация
  - Публикация
  - Получение статуса
  - Обработка ошибок
- Сначала можно поддержать ручной/полуавтоматический режим
- Автоматический режим включать только там, где он разрешен и стабилен

---

## P3. Управление объявлениями

### 13. Единый список объявлений
- Показывать все объявления кабинета
- Фильтры:
  - Проект
  - Аккаунт Авито
  - Категория
  - Город
  - Статус
  - Ответственный менеджер
- Статусы:
  - Черновик
  - На публикации
  - Активно
  - Отклонено
  - Архивировано
  - Ошибка
- Добавить быстрый переход к задаче, которая создала объявление

### 14. Редактирование объявлений
- Открытие карточки объявления
- Редактирование текста, цены, фото, адреса, опций
- История изменений
- Повторная отправка на публикацию
- Массовое редактирование выбранных объявлений

### 15. Шаблоны объявлений
- Создать шаблоны заголовков
- Создать шаблоны описаний
- Поддержать переменные:
  - Название товара
  - Город
  - Цена
  - Характеристики
  - Контакт
- Проверять, что после подстановки текст валиден

---

## P4. Массовое создание объявлений вручную

### 16. Каталог товаров/услуг
- Создать сущность "Товар / Услуга"
- Хранить:
  - Название
  - Описание
  - Фото
  - Цена
  - Себестоимость
  - Остаток
  - Характеристики
  - Категория Авито
- Связывать один товар с несколькими объявлениями

### 17. Импорт данных
- Импорт из CSV/XLSX
- Предпросмотр перед импортом
- Маппинг колонок
- Валидация строк
- Отчет об ошибках
- Частичный импорт валидных строк

### 18. Массовая подготовка объявлений
- Создание объявлений из каталога
- Создание объявлений из файла
- Массовый выбор города, аккаунта, категории
- Массовое применение шаблонов
- Таблица предпросмотра
- Пакетная публикация

---

## P5. Аналитика и экономика

### 19. Базовая аналитика объявлений
- Просмотры
- Контакты
- Добавления в избранное
- Конверсия из просмотра в контакт
- Динамика по дням
- Сравнение объявлений между собой

### 20. Финансовая аналитика
- Расходы на продвижение
- Стоимость контакта
- Стоимость лида
- Стоимость продажи, если продажа фиксируется
- ROI по объявлению
- ROI по проекту
- ROI по аккаунту Авито

### 21. Рекомендации по улучшению
- Много просмотров, мало контактов → проверить цену/описание
- Мало просмотров → проверить заголовок/фото/категорию/продвижение
- Много избранного, мало продаж → проверить цену или условия
- Объявление хуже среднего по проекту → предложить заменить фото/текст

## P6. Командная работа менеджеров

### 22. Назначение ответственных
- Назначать менеджера на проект
- Назначать менеджера на задачу
- Назначать менеджера на объявление
- Назначать менеджера на чат

### 23. Контроль работы менеджеров
- Количество обработанных чатов
- Среднее время первого ответа
- Пропущенные сообщения
- Просроченные задачи
- Количество созданных/отредактированных объявлений
- Эффективность объявлений по менеджеру

### 24. Внутренние комментарии
- Комментарии к задаче
- Комментарии к объявлению
- Упоминания менеджеров
- История обсуждений внутри кабинета

---

## P7. Чаты и лиды

### 25. Единый inbox
- Все чаты со всех аккаунтов в одном месте
- Фильтры:
  - Аккаунт
  - Проект
  - Объявление
  - Менеджер
  - Статус
- Статусы диалога:
  - Новый
  - В работе
  - Ожидает клиента
  - Требует внимания
  - Закрыт

### 26. Карточка лида
- Имя клиента
- Контакты, если доступны
- История сообщений
- Интересующее объявление
- Ответственный менеджер
- Статус лида
- Заметки
- Следующее действие

### 27. Шаблоны ответов
- Быстрые ответы
- Категории шаблонов
- Переменные в шаблонах
- Личные и общие шаблоны
- Статистика использования шаблонов

### 28. SLA по чатам
- Настроить лимит времени первого ответа
- Подсвечивать просроченные чаты
- Уведомлять ответственного менеджера
- Уведомлять администратора при долгом игноре

## P8. Автоматизация и правила

### 29. Конструктор правил
- Триггеры:
  - Создано объявление
  - Объявление отклонено
  - Пришло сообщение
  - Нет контактов N дней
  - Расход превысил лимит
  - Остаток товара закончился
- Действия:
  - Уведомить менеджера
  - Поставить задачу
  - Остановить продвижение
  - Снять объявление
  - Изменить статус
  - Предложить рекомендацию

### 30. Безопасный режим автоматизации
- Режим "только рекомендация"
- Режим "действие после подтверждения"
- Режим "автоматически"
- Лимиты на автоматические действия
- Журнал всех автоматических действий

---

## P9. Продвижение и ставки

### 31. Ручное управление продвижением
- Список объявлений с активным продвижением
- Включение/выключение продвижения
- Настройка бюджета
- Настройка периода
- История расходов

### 32. Автоматические стратегии
- Стратегия "не превышать дневной бюджет"
- Стратегия "остановить неэффективные объявления"
- Стратегия "усиливать объявления с хорошей конверсией"
- Стратегия "держать стоимость контакта ниже X"
- Обязательный предпросмотр действий перед включением стратегии

## P10. Репутация, отзывы, отчеты

### 33. Отзывы и рейтинг
- Сбор отзывов по аккаунтам
- Уведомления о новых отзывах
- Отдельный список негативных отзывов
- Шаблоны ответов
- Контроль ответа менеджера
- Динамика рейтинга

### 34. Отчеты
- Еженедельный отчет по проекту
- Отчет по менеджерам
- Отчет по расходам
- Отчет по объявлениям
- Отчет для владельца бизнеса
- Экспорт в XLSX/PDF

### 35. Агентский режим
- Несколько клиентов внутри одного аккаунта сервиса
- Разделение кабинетов по клиентам
- Отчеты для клиентов
- Ограничение доступа сотрудников агентства
- История действий по каждому клиенту
